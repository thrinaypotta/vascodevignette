# Vasco De Vignette — Photography Portfolio

A story-driven photography portfolio that **auto-updates from Google Drive**.  
Add photos and edit text in Drive → site rebuilds automatically every 24 hours.

---

## How it works

```
Google Drive folder  →  GitHub Action (runs daily)  →  data.json  →  Website
```

You manage everything from Google Drive. No coding needed after setup.

---

## One-time Setup (30 minutes total)

### Step 1 — Google Drive folder structure

Create this exact folder structure in your Google Drive:

```
📁 VdV-Portfolio/
  📁 Zugspitze/
      info.json
      photo1.jpg
      photo2.jpg
  📁 Schwerin/
      info.json
      photo1.jpg
  📁 Rugen/
      info.json
      hero-photo.jpg
      photo1.jpg
```

**Rules:**
- One subfolder per place
- Each folder needs an `info.json` (see template below)
- Add as many `.jpg` / `.webp` photos as you like (max 9 shown)
- Name the hero image `hero-anything.jpg` — it becomes the background

---

### Step 2 — Create info.json for each place

Copy this template into each place folder as `info.json`:

```json
{
  "name": "Zugspitze",
  "region": "Bavaria · 2,962m",
  "tagline": "Germany's highest peak",
  "subtitle": "& the edge of light",
  "coords": "47°25'16\"N  10°59'07\"E",
  "storyTitle": "As one world sets, the other arises.",
  "story": [
    "First paragraph of your story.",
    "Second paragraph here."
  ],
  "camera": "Fujifilm X · Classic Chrome",
  "season": "Winter 2026",
  "theme": "Landscape",
  "instagramUrl": "https://www.instagram.com/p/YOURPOSTID/",
  "order": 1
}
```

The `order` number controls the sequence on the website (1 = first).

---

### Step 3 — Make photos publicly accessible

For each photo in Drive:
1. Right-click the photo → Share → Change to **Anyone with the link**
2. (You only need to do this for the folder, not each file individually)

**Easier method:** Right-click the top-level `VdV-Portfolio` folder → Share → **Anyone with the link** → Viewer. This covers everything inside.

---

### Step 4 — Create a Google Service Account

This lets GitHub read your Drive automatically.

1. Go to [console.cloud.google.com](https://console.cloud.google.com)
2. Create a new project (name it anything, e.g. "vdv-portfolio")
3. Enable the **Google Drive API**:
   - APIs & Services → Enable APIs → search "Google Drive API" → Enable
4. Create a Service Account:
   - APIs & Services → Credentials → Create Credentials → Service Account
   - Name it anything → Create → Done
5. Download the JSON key:
   - Click the service account → Keys tab → Add Key → Create new key → JSON
   - Save the downloaded `.json` file — you'll need it in the next step
6. Share your Drive folder with the service account:
   - Open the JSON file — find the `"client_email"` field (looks like `name@project.iam.gserviceaccount.com`)
   - In Google Drive, right-click `VdV-Portfolio` → Share → paste that email → Viewer → Send

---

### Step 5 — Add the secret to GitHub

1. Go to your GitHub repo → **Settings** → **Secrets and variables** → **Actions**
2. Click **New repository secret**
3. Name: `GOOGLE_SERVICE_ACCOUNT_JSON`
4. Value: paste the **entire contents** of the JSON key file you downloaded
5. Click Add secret

---

### Step 6 — Upload these files to GitHub

Upload all these files to your repo (keeping the folder structure):

```
index.html
data.json
scripts/build.py
.github/workflows/update.yml
README.md
```

Enable GitHub Pages: **Settings → Pages → Deploy from branch → main → / (root)**

## Day-to-day use

### Adding a new place
1. Create a new folder in `VdV-Portfolio/` in Google Drive
2. Add an `info.json` with your story
3. Drop in your photos
4. Wait until 6am UTC — site updates automatically
5. Or: go to GitHub → Actions → "Rebuild portfolio" → Run workflow (instant)

### Updating a story
1. Open `info.json` in the place folder in Google Drive
2. Edit the text
3. Save — next rebuild picks it up

### Adding photos to an existing place
1. Drop photos into the place folder in Drive
2. Done — next rebuild adds them

### Changing photo order
Name photos alphabetically in the order you want:
`01-main.jpg`, `02-detail.jpg`, `03-closing.jpg`

---

## Manual trigger

Go to **GitHub → Actions tab → "Rebuild portfolio from Google Drive" → Run workflow**

This rebuilds immediately without waiting for the daily schedule.

## File reference

| File | Purpose |
|------|---------|
| `index.html` | The website — reads `data.json` to display everything |
| `data.json` | Auto-generated by build script — don't edit manually |
| `scripts/build.py` | Reads Drive, generates `data.json` |
| `.github/workflows/update.yml` | Schedules the daily rebuild |
